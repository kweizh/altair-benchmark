**What the script does (layer by layer)**

1. **Row-limit guard** — `alt.data_transformers.disable_max_rows()` is called before any chart construction so the large remote dataset is not silently truncated.

2. **Data source** — `data.unemployment_across_industries.url` is passed directly to each `alt.Chart(source)` call; the CDN URL is baked into the Vega-Lite spec, requiring no local file at render time.

3. **Area layer** — `mark_area(interpolate="monotone")` with `yearmonth(date):T` on X, `sum(count):Q` with `stack="center"` on Y (axis hidden), and `series:N` on Color with `scheme="category20b"`. The hover selection is attached here via `add_params(hover)`.

4. **Hover selection** — `alt.selection_point(name="hover", on="pointerover", nearest=True, encodings=["x"], empty=False)` — fires only when the pointer is near an actual data point on the X axis.

5. **Rule layer** — `mark_rule` whose `opacity` channel uses `alt.condition(hover, alt.value(0.85), alt.value(0))`, making the white vertical rule visible only at the hovered month.

6. **Text annotation layer** — `mark_text` with two transforms:
   - `transform_window` ranks rows within each `date` group by `count` descending (`op: "rank"`).
   - `transform_filter("datum.date_rank === 1")` keeps only the top-ranked (dominant) industry per month.
   The `text` and `opacity` encodings are both conditioned on the hover param so the label appears only while hovering.

7. **Composition** — `alt.layer(area, rule, annotation)` combines the three layers; the shared data source keeps the X scale perfectly aligned across all layers.
